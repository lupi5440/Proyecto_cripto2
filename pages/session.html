Gestión Segura de Sesiones

Las sesiones permiten mantener el estado de un usuario autenticado. Una gestión inadecuada puede llevar a secuestro de sesión, fijación de sesión, y otros ataques.

Ataques Comunes:

  • Session Hijacking: Robo del ID de sesión (ej. por XSS o sniffing)
  • Session Fixation: El atacante fija un ID de sesión conocido
  • Session Replay: Reutilización de un ID de sesión capturado

Defensa en Profundidad:

Nivel 1 - Cookies Seguras:

  • HttpOnly: Previene acceso por JavaScript
  • Secure: Solo transmisión por HTTPS
  • SameSite: Previene CSRF

Nivel 2 - Gestión del Servidor:

  • IDs de sesión aleatorios y criptográficamente seguros
  • Regenerar ID después del login
  • Timeout de inactividad
  • Almacenar mínima información en cookies

Nivel 3 - Monitoreo:

  • Detectar cambios de IP/User-Agent
  • Limitar sesiones concurrentes
  • Logging de actividad sospechosa
